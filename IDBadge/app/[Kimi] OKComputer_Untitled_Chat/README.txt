# Kimi's BFI Landing Page - Run Instructions

## Overview
This is a React/TypeScript landing page for BFI Access Cards, generated by Kimi AI. It features 3D scroll animations using GSAP and ScrollTrigger, scroll-spy navigation, and a tabbed interface for onboarding/offboarding documentation.

## Required Tools

1. **Node.js** (>= 18.x)
2. **npm** (>= 8.x)
3. **Current: PowerShell** (running on Windows)

## Quick Start Steps

### 1. Navigate to the Project Directory
Run these commands in PowerShell:


### 3. Install Dependencies
```powershell

```

### 4. Build the Project
```powershell

```

### 5. Start the Development Server
```powershell

```

### 6. Access the Landing Page
Open your browser and go to:
```
http://localhost:3000
```

## What the Landing Page Features

### Main Components
- **Navigation** - Scroll-spy active link highlighting
- **Hero Section** - Featured card with 3D scroll animation (rotateY 0→200°)
- **Process Section** - Tabbed interface for:
  - On-Boarding
  - Off-Boarding
  - Documents
  - Overview
- **Footer** - Standard corporate footer

### Animation Features
- **3D Card Scroll Rotation** - GSAP + ScrollTrigger with scrub:1
  - Card rotates along Y-axis, scales, and moves during scroll
  - Timeline: rotateY 0→180° (0-40%), then rotateY 180→200°, rotateX 0→5° (40-100%)
- **Card Entrance** - Single GSAP tween on load
- **Section Entrance** - IntersectionObserver with CSS animations
- **Tab Switching** - CSS transitions for fade-in effects

## Component Architecture

### Tech Stack
- **Framework**: React + TypeScript
- **Build Tool**: Vite
- **Animation**: GSAP + ScrollTrigger (GSAP v3.15.0)
- **Icons**: Lucide React
- **Styling**: Custom CSS (no shadcn/ui)
- **Navigation**: IntersectionObserver-based scroll-spy

### Key Tech Specs
- Single-page application
- No additional UI framework
- Pure client-side tab state management
- Cross-origin design avoids component library conflicts
- Custom corporate styling (navy/gold palette)

## Troubleshooting

### If npm install fails:
1. Delete node_modules and package-lock.json:
```powershell
Remove-Item -Path "node_modules", "package-lock.json" -Recurse -Force
```

2. Try installing again:
```powershell
npm install
```

### If build fails:
Check for missing dependencies:
1. Ensure all required packages in package.json are available
2. Try removing node_modules and package-lock.json completely

## Project Notes

- Download cards reference non-existent PDF files (buttons trigger no download)
- All animations are either scroll-driven (GSAP) or simple CSS transitions
- Uses a single IntersectionObserver for both navbar tracking and section reveals
